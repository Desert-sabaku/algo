# 再帰関数の実装tips

## 最小構成（`n=0, 1`）のときを決めろ

ここが曖昧だと無限再帰になってしまう。

ハノイの塔なら円盤が一枚の時どうなるのか。
階乗なら0!と1!はどうなるのか。

## 分割

再帰は、「自分と同じ関数を、より小さな入力で呼び出す」ことである。
毎回入力が確実に小さくなるか確認する。

## 一段分の処理を言葉に起こす

先に自然言語で処理の手順を疑似的に記述する。
例えば、ハノイの塔なら「一番上の円盤を、補助の柱に移動する」といった具合。

## 責務は一つ

一般論だが、一つの関数には一つの責務。

## 小さい値で処理を追う

再帰関数を実装したら、まずは小さい値で処理を追う。
紙でもいいし、デバッガーを使ってもいい。

# 失敗等

- ベースケースがない・到達しない
- 入力が減っていない
- 返り値の結合順
- グローバル変数を更新してしまい、2度目以降に壊れる
